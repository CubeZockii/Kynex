ModBot Documentation

Your guide to using and hosting the powerful ModBot for Discord

ModBot is a powerful, flexible, and easy-to-use moderation bot designed to help keep your Discord server safe and well-managed. It includes a variety of essential moderation tools and can be self-hosted for full control and customization.

Getting Started

To invite the hosted ModBot to your server, use this link:

➕ Invite ModBot

Commands & Features

  • Ban / Unban – Permanently remove troublemakers
  • Kick – Temporarily remove users from the server
  • Purge – Bulk delete messages from a channel
  • Warn / Unwarn – Issue warnings to users (with automatic punishment)
  • Warnings – View a user's current warnings
  • Mute / Unmute – Temporarily silence a user
  • Lock / Unlock – Prevent or allow messages in a channel
  • Role – Add or remove roles from users
  • Help – Display a list of all commands
  • Ping – Check the bot’s response time
  • Revive – Ping users in a quiet chat to re-engage
  • Logging (optional) – Track bans, mutes, deletes and more

Automatic Punishment System

Warnings are tracked, and if a member reaches a configured threshold, the bot can automatically apply punishments like muting or banning. This system can be customized in the bot’s code to suit your moderation policy.

Logging System (Optional)

If the logging system is not needed, you may encounter this warning when starting the bot:

Self-Hosting Instructions

  1. Download the source files: Download ZIP
  2. Extract the ZIP and open a terminal in the folder
  3. Install dependencies with: pip install -r requirements.txt
  4. Create a .env or .json file with your bot token and other config. (if needed)
  5. I recommend a .json file because the bot is coded with .json
  6. Run the bot with: python bot.py

For best results, host the bot using a cloud platform like KataBump or SillyDev.
